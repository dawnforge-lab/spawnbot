# Code Style and Conventions

## Language & Module System
- **Plain JavaScript** (no TypeScript, no JSDoc types)
- **ES Modules** (`import`/`export`, `"type": "module"` in package.json)
- Uses `fileURLToPath(import.meta.url)` for __dirname equivalent

## Naming Conventions
- **camelCase** for variables, functions, methods, properties
- **PascalCase** for classes (`Daemon`, `McpServer`, `InputQueue`, `WireClient`)
- **UPPER_SNAKE_CASE** for constants (`MAX_QUEUE_SIZE`, `PRIORITY`, `STATES`, `DEFAULT_TIMEOUT`)
- **snake_case** for SQL column names (mapped from camelCase JS via Drizzle)
- **kebab-case** for file and directory names (`base-server.js`, `poller-manager.js`)

## Class Style
- Standard ES6 classes with constructor
- Private methods prefixed with underscore (`_handleLine`, `_send`, `_startWire`)
- No getter/setter abuse — plain methods

## Code Patterns
- Destructuring in constructor parameters: `constructor({ name, version = '0.1.0' })`
- Console.log with prefix tags: `console.log(\`[QUEUE] ...\`)`
- Spread for immutable copies: `{ ...item, priorityNum: priority }`
- `??` for nullish coalescing, no `||` for defaults where 0/false matter
- Template literals for string interpolation

## No Linter / Formatter Configured
- No ESLint, Prettier, or similar tooling in package.json
- Follow existing code style by convention

## Error Handling
- **CRITICAL: No fallbacks** — transparent errors that can be fixed, not hidden
- Let errors propagate visibly rather than swallowing them

## Config Files
- YAML for human-readable config (SOUL, CRONS, integrations)
- SQLite for operational state
- .env for secrets/credentials
- All config files generated by `spawnbot setup`, gitignored
